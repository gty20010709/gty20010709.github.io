我好想找不出更多了，日后想到再补充。

三、结果

我先在备用机上重装系统，缓解一下心中的骚动，主力军要办公，还是得Windows。

Crossref:开放的论文元数据检索API

零、 缘起

本需要研究从 Google Scholar 和 百度学术 抓取数据，但无奈人机验证和条目摘要难以提取，故寻找其他途径。在浏览 Sci-Hub 时， 从其 [database页面](https://sci-hub.se/database) 发现了 Crossref, 它提供开放的 API。

此是为缘起。

一、从 Corssref 检索文献

Crossref 是一个非盈利组织，提供文献元数据的检索服务，并提供免费的 API 服务，不需要登录便可以获得 50 次/s 的 API 调用权限。

Crossref 的 API 文档位于： [https://api.crossref.org/swagger-ui/index.html#/Works/get_works。

通过 Crossref 的 API， 可以凭关键词进行搜索，取得指定数目的文献信息，每条信息中，包含论文的标题、作者、URL、DOI、摘要（部分有）等信息。元数据示例

Crossref 的所有数据都可以从这里下载： https://academictorrents.com/checkb.htm?search=Crossref

Crossref 的不足

  • 中文文献少

二、Google Scholar 与 百度学术

在二者的网页进行搜索后，可以得到一个论文列表，在列表的每一行中，是论文的标题、作者、不完整的摘要等信息。

在 Google Scholar 上点击论文标题，会离开 Google 的网站，跳转到论文提供者的页面或者是直接下载 PDF，要获取完整的论文摘要，是十分困难的。

在百度学术上点击论文的标题，会进入一个详情页面，页面包含该论文的摘要。但使用程序在百度学术上搜索时，会频繁触发人机验证，暂未能解决；Google Scholar 也会触发人机验证，未能解决。

如何在Vercel中使用自定义域名

〇、 缘起

关于这个问题，许多人已经写了文章 ———— 我在尝试之初，便是看着那些文章进行的。那么，为什么我还要写这篇文章呢？

在 Vercel 中使用自定义域名，步骤并不复杂，可以抽象为： 在 Vercel 的面板中输入自己的域名 –> 在 DNS 解析商添加一条 CNAME 记录 –> 耐心等待一会儿。

我想就我在其中遇到的几个要点做个记录，便写了这篇文章。

一、 域名问题

一开始， 在 freenom 可以白嫖免费域名， 但现在不行了；现在我在使用 namesilo 所出售的域名，关于域名和 VPS 可以上网搜“瓦力箱子”这个网站。

PS： namesilo 支持 支付宝 ( Alipay ) 付款。

买完域名后，最好把域名的 nameserver 切换为 Cloudflare，可以在 https://dash.cloudflare.com/ 进行操作，在 Cloudflare 面板的顶部，点击 “Add site”， 再跟着指引走便可。

二、 DNS 解析问题

Cloudflare 可以对解析的网址进行代理，我第一次尝试使用自定义域名时，非常迷惑是否需要开启代理，经过多次尝试，我发现不开代理的时候，网站可以正常访问；反之则不行。

最近， .ml 域名突然不能用了，我又新买了一个 .top 的域名。在为托管在 Vercel 上的网站添加自定义域名时，我发现，在 Cloudflare 添加 CNAME 解析记录的时候，开启 Cloudflare 的代理也是可以用的。

三、 其他

经过实践测试，还需要保证 “SSL/TLS encryption mode” 为 “Full”， 可以在 Cloudflare 对应域名那里设置。
