如何在Vercel中使用自定义域名

〇、 缘起

关于这个问题，许多人已经写了文章 ———— 我在尝试之初，便是看着那些文章进行的。那么，为什么我还要写这篇文章呢？

在 Vercel 中使用自定义域名，步骤并不复杂，可以抽象为： 在 Vercel 的面板中输入自己的域名 –> 在 DNS 解析商添加一条 CNAME 记录 –> 耐心等待一会儿。

我想就我在其中遇到的几个要点做个记录，便写了这篇文章。

一、 域名问题

一开始， 在 freenom 可以白嫖免费域名， 但现在不行了；现在我在使用 namesilo 所出售的域名，关于域名和 VPS 可以上网搜“瓦力箱子”这个网站。

PS： namesilo 支持 支付宝 ( Alipay ) 付款。

买完域名后，最好把域名的 nameserver 切换为 Cloudflare，可以在 https://dash.cloudflare.com/ 进行操作，在 Cloudflare 面板的顶部，点击 “Add site”， 再跟着指引走便可。

二、 DNS 解析问题

Cloudflare 可以对解析的网址进行代理，我第一次尝试使用自定义域名时，非常迷惑是否需要开启代理，经过多次尝试，我发现不开代理的时候，网站可以正常访问；反之则不行。

最近， .ml 域名突然不能用了，我又新买了一个 .top 的域名。在为托管在 Vercel 上的网站添加自定义域名时，我发现，在 Cloudflare 添加 CNAME 解析记录的时候，开启 Cloudflare 的代理也是可以用的。

三、 其他

经过实践测试，还需要保证 “SSL/TLS encryption mode” 为 “Full”， 可以在 Cloudflare 对应域名那里设置。

如何用Python获取本机的IPV4地址

〇、 缘起

在使用 Flask 程序在局域网内提供服务时，直接将 host 设置为 0.0.0.0 可能并不是最佳的方法。

假设，我运行了两个服务：

- 在 `5000` 端口运行了一个 网页服务；

- 在 `5001`  端口运行了一个带有 web 界面的文件共享服务；

在网服务中，有一个按钮导向文件共享服务。网页服务监听 0.0.0.0 这个地址，是没有问题的；但为了能正确跳转到文件共享服务，我们需要获取到 本机的 IP 地址 ，然后，手动写入网页模板中。

这会引发一个问题，当我们切换部署的机器时，还需要手动更改网页模板中的 IP 地址。

为了解决这个问题，本文尝试在 Python 程序中自动获取本寂的 IP 地址。

此是为本文之缘起。

一、 代码及注释

下面是对每一行的详细解释：

import socket

这行代码导入了 Python 的 socket ↗ 模块，这个模块提供了对低级网络服务的访问。

def get_ip() -> str:

这行代码定义了一个名为 get_ip 的函数，该函数的返回类型被注解为字符串 (str)。

s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

这行代码创建了一个新的 socket 对象 ↗。AF_INET 是地址族，指定了使用 IPv4 地址或主机名。SOCK_DGRAM 是套接字类型，指定了使用 UDP 协议。

s.connect(("8.8.8.8", 80))

这行代码尝试将创建的套接字连接到指定的 IP 地址和端口。这里的 IP 地址 “8.8.8.8” 是 Google 提供的公共 DNS 服务器地址。端口号 “80” 是 HTTP 服务的默认端口。这里的 connect 只是为了获取本地 IP，并不发送任何数据。

return s.getsockname()[0]

最后这行代码使用 getsockname 方法获取当前套接字的地址。这个方法返回一个元组，包含主机地址和端口号。在这里，我们只关心主机地址（即你的 IP 地址），所以使用 [0] 索引获取第一个元素。
