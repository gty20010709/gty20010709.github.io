自建邮箱服务之记录

〇、 缘起

因为参与公益项目，我需要去建立一个邮箱服务器，做一个尝试。这是我尝试过后的笔记和总结。

一、 选取邮箱

我首先 Google 到了这篇文章： Best Open Source Email Servers，里面谈到了12个开源的邮箱服务软件，我又补充了我查阅资料遇到的，总结如下：

名称 描述 网址
Postal 专注于传出邮件的邮件服务器 https://github.com/postalserver/postal
mailcow 完整的邮件服务器套件 https://mailcow.email/
Cuttlefish 简单的事务邮件服务器 https://cuttlefish.io/
Apache James 企业级开源邮件服务器 https://james.apache.org/
Haraka 使用 Node.js 构建的 SMTP 服务器 https://haraka.github.io/
Modoboa 全能的邮件服务器 https://modoboa.org/en/
Postfix 邮件传输代理 https://www.postfix.org/
Maddy 轻量级全能邮件服务器 https://maddy.email/
Dovecot IMAP 服务器 https://www.dovecot.org/
Poste.io 基于多组件的邮件服务器解决方案 https://poste.io/
iRedMail 搭建邮件服务器的工具 https://www.iredmail.org/
Mailu 基于 Docker 的简单邮件服务器 https://mailu.io/2.0/
Mox 一个现代的开源邮件服务器,提供自托管、安全、易用的电子邮件服务。 https://github.com/mjl-/mox
Salmon 一个 Python 库，用于实现邮件服务器，灵活性最大。 https://salmon-mail.readthedocs.io/en/latest/
Mail-in-a-Box 这个软件提供了DNS自动配置，免去了新手在面对各种 DNS 记录时的混乱 https://mailinabox.email/

我结合自己的条件（VPS性能）和部署的方便程度，以及需求（能接受，可通过程序批量注册）等，最终选取了 Maddy。

二、 Maddy 的安装配置

1. 前提

  • 1 台 VPS （1core, 1g, 5g disk）(IP地址为： 1.2.3.4)
  • 2 个域名 (example.org, mx.example.org) 及对应的 ssl 证书。
  • 1 个大脑

2. 安装和启动 Maddy

我是照着 官方教程 进行的

安装方式有 docker、 pre-build bin、 from source等（如果服务器装的是 Arch， 还可以通过 aur 来安装）。我选择的是 pre-build bin，即下载官方build好的二进制文件。

从 Github release 下载 maddy-版本-x86_64-linux-musl.tar.zst，用 unzstd 解压。

为了安全， Maddy 不会以 root 身份运行，我们需要为 maddy 创建一个用户：useradd -mrU -s /sbin/nologin -d /var/lib/maddy -c "maddy mail server" maddy 。

在启动前，我们需要填写配置文件，告诉maddy我们的域名和证书所在位置。假设前面提到的两个域名证书都放在了 /etc/maddy/cert/[域名]/{fullchain.pem, privkey.pem}。

（在不考虑安全的情况下， 其实只需要 mx.example.org 这个域名的证书就可以了。）

3. DNS 配置

在开始之前，了解下面两个话题，会有助于我们对过程的理解：

  • 电子邮件是如何工作的？
  • 邮件服务器相关概念
  • Cloudflare Blog：应对电子邮件欺骗和网络钓鱼

如果只需要基本的收发服务，而完全不考虑安全，那么配置三条 DNS 记录就可以了：

处于安全考虑，我们还需要配置：

  • 发件人策略框架 (SPF)：列出可以代表您的域发送电子邮件的授权 IP 地址和域。
  • 域名密钥识别邮件 (DKIM)：通过对电子邮件进行加密签名来确保电子邮件的真实性。
  • 基于域的消息身份验证报告和一致性 (DMARC)：接收有关您的电子邮件流量的汇总报告，并提供有关电子邮件接收者应如何处理不合格电子邮件的明确说明。

一般配置到这里就够了，等待 DNS 记录生效，邮箱就可以注册了。为了更进一步的安全，还可以设置 MTA-STS 和 DANE，我第一次配置了，之后发现并不是很必要，这里就不深入介绍，想配置可以参照官方文档: https://maddy.email/tutorials/setting-up

四、 注册邮箱

在使用之前，还需要注册，在VPS上通过命令就可以完成。通过 VPS 命令管理邮箱的好处在于，我可以通过写 Python 脚本，来实现一个远程注册和删除账户的功能（使用 FastAPI 来提供服务），这是后话了。

现在，就可以正式使用邮箱了。
