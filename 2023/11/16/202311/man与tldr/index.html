man与tldr

〇、 缘起

今天早上在吃饭时，我刷了两个视频：
- 4 Weird (& Less-Used) Built-In Python Data Structures & When To Use Them
- 5 Awesome Linux Terminal Tools You Must Know

第一个视频讲了 Python 中的 frozenset, defaultdict, deque, namedtuple，基本在我的知识范围内，没啥好记录的；第二个视频中，介绍了几个在 Linux 命令行中的实用工具，我印象最深刻的是用于文件搜索的 fzf 和 用于查看命令用例的 tldr。

tldr, 我觉得可以在一定程度上取代 man 命令，因此写文记录。此是为缘起。

一、 man

Linux 上的 man 命令是 manual （手册）的缩写。假如我对 fzf 命令不够了解，使用 man fzf 去查看手册，大量的文本往往会让人看得头大，一时半会也不知道如何使用命令。但是，如果有一些直观的使用示例和说明，我们是不是一下就能理解 fzf 可以干什么？这其实就是 tldr 所做的事情。

二、 tldr

在英文互联网中，你可能会经常看到 TLDR 这几个字母，它是Too Long; Didn't Read 的缩写，表示太长未读或太长不想读。tldr 这个命令的名字就来源于此。

如果我想看看 fzf 命令有哪些常用的用法， 可以输入tldr fzf。

从下面的输出，我们可以看到。

  • fzf 可以与 find 结合使用， 搜索指定目录下的所有文件；
  • 与 ps 结合使用，搜索正在运行的进程；
  • 可以将符合条件的多个文件的内容汇总到一个文件（cat命令也能做到）
  • 可以指定条件进行查询
  • …

四、 cmd –help

有人可能会说，用 --help 参数，查看命令行程序自带的帮助信息，也比 man 简单呀。实际上，对大多数人来说，程序自带的帮助信息也不一定能读得懂，不如直观的例子来得明白。

比如 fzf --help，依旧很抽象。

最后

tldr 或许是快节奏社会节奏的产物，很多时候，静心去读手册（manul）是难得乃至不可能的事情了。

tldr 只是辅助快速上手一个工具，真正的掌握，还是要基于对手册的理解和精读 + 练习 来实现的。手册博大精深，很多时候不仅能学到一个命令的使用，还能学到一个命令的设计思想、历史沿革等等。

PS: make 的手册，pdf档有200多页。 GNU make manual
